Detailed Description

Base class for writing asynchronous OR synchronous log messages. For asynchronous logging, one must call Close/Dispose upon application exit or application will NEVER exit as messages are lazily written by a low-priority foreground thread. The foreground thread is necessary to be able to allow time to flush all queued messages before the close operation completes. Upon application exit, a background thread will simply terminate and outstanding queued messages will be lost.
